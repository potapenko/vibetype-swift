Use https://github.com/potapenko/spec-first-bootstrap as the spec-first
reference.

This is an existing Swift project with incomplete or unreliable documentation.

First analyze the codebase and identify the main product areas.

Then prepare a structured clarification pass for the user or team.

The clarification should focus on:

- intended microphone and transcription behavior
- text output and handoff expectations
- privacy, permissions, and persistence rules
- edge cases and failure policy
- known exceptions
- places where current code may not reflect intended product behavior

After that, recommend which specs should be created first and which areas can
wait.

Do not implement anything yet.
